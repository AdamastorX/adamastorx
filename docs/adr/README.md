# Architecture Decision Records

One ADR per significant, hard-to-reverse decision: new tool, changed
boundary, rejected alternative worth remembering. Not for routine
implementation choices.

Format: `NNNN-short-title.md`, numbered sequentially, never renumbered or
deleted — superseded ADRs stay, marked `Status: Superseded by NNNN`.

Each ADR: Context, Decision, Consequences. Keep it to half a page.

This folder is the single place decisions live — there is no separate
"decisions" folder. If it changed how the system is built and future-you
would ask "why did we do it this way", it's an ADR.
